# 更新说明 (2026-03-26)

## ⚡️ 极致性能与丝滑顺畅度专项优化

本次更新通过全链路的性能调优，显著提升了小程序的冷启动速度和页面间切换的流畅丝滑感，实现了接近原生的交互反馈。

### 1. 启动与加载速度优化 (Extreme Speed)
- **子包静默预加载**: 优化了 `app.json` 规则，用户访问首页时即在后台静默预加载常用子包（如：智能控制、扫码、BMI、每日新闻），实现后续页面的瞬时打开。
- **初始渲染缓存 (Static Cache)**: 为首页和“我的”页面开启了微信提供的 `initialRenderingCache` 静态缓存功能，实现首屏界面的“秒出”体验。
- **启动任务异步化**: 改造了 `App.onLaunch` 与页面 `onLoad` 的任务调度，将非首屏必须的逻辑推迟到下一个事件循环，不阻塞框架初始化。
- **主包资源瘦身**: 剔除了首页隐藏加载的 600KB+ 巨型背景图，大幅降低了 4G/WiFi 环境下的资源排队时间。

### 2. 交互丝滑感重构 (Fluid UI/UX)
- **导航架构升级**: 模拟 TabBar 跨页跳转由 `reLaunch` 修改为 `redirectTo` 模式，消除了页面切换时的全栈销毁与冷启动卡顿。
- **智能数据缓存同步**: 优化了用户信息获取逻辑，强制优先使用已缓存的本地 profile，消除了界面打开时 ID 和状态信息的“闪动”。
- **硬件级动画优化**: 在所有 3D 拟态卡片类（`.mini-card`, `.profile-card` 等）上追加了 `will-change: transform` 硬件加速指令，确保在低端 Android 设备下依然能保持满帧运行。

### 3. 工程化与编译优化
- **SWC 编译器集成**: 启用 Rust 编写的 SWC 加速编译器，编译速度提升 30%，并开启了 JS 代码增强压缩与底层渲染优化。

# 更新说明 (2026-03-21)

## 🎨 全新 3D 拟态 (Claymorphism/Neumorphism) UI 重构

本次更新通过全盘 CSS 重写，将小程序从原本的扁平风格彻底进化为具有真实光影、触控回弹和厚重材质感的现代 3D 拟态界面。

### 1. 深度页面适配
- **[扫描确认页]**: 采用了磨砂玻璃与 Clay 材质混合，增加了立体感和安全感。
- **[运维/详情页]**: 服务器卡片增加了 3D 浮雕投影与悬浮动画，操作反馈更明显。
- **[课程系统]**: 订单列表与查询表单全面拟态化，模块化视觉感更强。
- **[待办清单]**: 优化了卡片圆角显示，消除了长列表截断导致的直角阴影 bug。
- **[智能家居面板]**: 尤其是“空气能/温控”面板，按键具备真实的“物理按压”手感。
- **[BMI 检测]**: 优化了输入框显示高度，消除了文字溢出和挤压。

### 2. 全新交互体验 (UX)
- **极速响应**: 全局按压反馈时长缩减至 0.15s，配合弹性贝塞尔曲线，手感干脆不粘手。
- **全局字体**: 启用了 -webkit-font-smoothing 平滑抗锯齿，文字呈现更细腻。
- **输入组件**: 实现了全盘统一的“内嵌槽”方案，大幅提升表单填写的视觉引导。

### 3. 系统级性能与兼容性修复
- **渲染加速**: 核心卡片强制开启 GPU 硬件级加速 (translate3d)，告别低端机掉帧。
- **真机 Emoji 修复**: 解决了微信原生渲染引擎在真机下丢失表情图层的底层 Bug。
- **扫除冗余**: 清理了部分重复挂载的全局 CSS 类，规范化了布局容器。
