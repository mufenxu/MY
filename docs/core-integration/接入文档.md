# 扫码登录接入文档 (外部开发者版)

本文档旨在帮助第三方网站接入 **[Mufenxu 扫码登录平台]**。

**API 接口地址**: `https://pxyb.cn/api/core`

---

## 接入前准备

1.  联系管理员，在管理后台注册您的网站。
2.  获取分配给您的 **App ID** (例如 `xxxxxxxx`)。
3.  确保您的网站可以通过公网访问（用于测试）。

---

## 快速接入 (Copy & Paste)

您可以直接参考 `docs/demo_client.html` 文件，或者将以下代码嵌入您的登录页面。

### 1. 引入必要文件
我们需要生成二维码，推荐使用 `qrcode.js`。
```html
<script src="https://cdnjs.cloudflare.com/ajax/libs/qrcodejs/1.0.0/qrcode.min.js"></script>
```

### 2. HTML 容器
```html
<div id="qrcode-container"></div>
<p id="tips">请使用微信扫码</p>
```

### 3. JavaScript 核心逻辑
将 `YOUR_APP_ID` 替换为您申请到的真实 ID。

> **注意**：以下是前端部分演示。在真实场景中，“换取 Token”这一步（Step 3）必须在您的**后端服务器**上进行，因为需要用到 **App Secret**，千万不能把 Secret 暴露在前端代码里。

```javascript
const API_BASE = 'https://pxyb.cn/api/core';
const YOUR_APP_ID = '替换为您的AppID'; 

let qrToken = '';

// Step 1: 初始化，获取二维码
function getNewQRCode() {
    // 从 sessionStorage 获取旧 Token，用于告知后端销毁旧码
    const oldToken = sessionStorage.getItem('my_app_qr_token');
    
    fetch(API_BASE + '/auth/qrcode/create', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ 
            appId: YOUR_APP_ID,
            oldToken: oldToken // 传入旧 Token
        })
    })
    .then(res => res.json())
    .then(data => {
        if(data.qrToken) {
            qrToken = data.qrToken;
            // 将新 Token 存入 session，防止刷新页面产生多余有效码
            sessionStorage.setItem('my_app_qr_token', qrToken);
            
            // 渲染二维码
            document.getElementById("qrcode-container").innerHTML = ""; // 清空旧码
            new QRCode(document.getElementById("qrcode-container"), data.qrCodeUrl);
            // 开始轮询状态
            if(window.statusTimer) clearInterval(window.statusTimer);
            window.statusTimer = setInterval(checkStatus, 2000);
        }
    });
}

getNewQRCode();

// Step 2: 轮询状态
function checkStatus() {
    if(!qrToken) return;
    
    fetch(`${API_BASE}/auth/qrcode/status?qrToken=${qrToken}`)
    .then(res => res.json())
    .then(data => {
        if(data.status === 'confirmed') {
            // 已确认登录，拿到临时CODE，发给您的后端
            sendToYourBackend(data.tempAuthCode);
        } else if (data.status === 'scanned') {
            document.getElementById('tips').innerText = '已扫码，请手机确认';
        }
    });
}

// Step 3: 您的后端服务器逻辑 (Node.js 示例)
/*

    const res = await axios.post('https://pxyb.cn/api/core/auth/token/exchange', {
        appId: 'YOUR_APP_ID',

        tempAuthCode: tempCode
    });

}
```

## 接口详情 (API Reference)

### 1. 创建二维码会话
`POST /auth/qrcode/create`
*   **参数**: 
    *   `appId`: (必填) 您的应用 ID
    *   `oldToken`: (建议) 上次申请的 Token，传入后服务器将立即作废旧码。
*   **返回**: `qrToken` (会话ID), `qrCodeUrl` (二维码内容), `expireIn` (有效期秒数)

### 2. 查询状态
`GET /auth/qrcode/status?qrToken=...`
*   **返回**:
    *   `waiting`: 等待扫码
    *   `scanned`: 已扫码，等待确认
    *   `confirmed`: 已确认，返回 `tempAuthCode`
    *   `expired`: 已过期

### 3. 换取凭证 (仅限服务端调用)
`POST /auth/token/exchange` (需要鉴权)
*   **参数**: `{"appId": "...", "secret": "...", "tempAuthCode": "..."}`
*   **返回**: `accessToken` (JWT), `user` (用户信息对象)

## 常见问题

Q: 接口报 403 App is disabled?
A: 请联系管理员在后台将您的应用状态设置为启用。

Q: 接口报 CORS 错误？
A: 目前服务器允许跨域。如果仍有问题，请联系管理员添加您的域名白名单。

## 进阶：如何与您现有的用户系统对接？(核心)

对于**第三方网站**，本平台只负责提供**“这个扫码的人是谁（OpenID）”**，不负责管理您网站里的权限。您需要自己在您的网站实现**“账号绑定”**逻辑。

### 推荐的绑定流程

**场景**：用户在您的网站（Site B）已经是管理员，希望以后能扫码登录。

1.  **用户登录**：用户先使用您网站原有的“账号+密码”登录。
2.  **绑定入口**：在“个人中心”提供一个 [绑定微信] 按钮。
3.  **发起绑定**：
    *   调用本平台 `create` 接口生成二维码。
    *   用户扫码。
    *   本平台返回 `{ user: { openid: "oCola7...", ... } }`。
4.  **关联数据**：
    *   **关键一步**：您将获得的 `openid` 保存到您自己数据库的用户表中（例如 `users` 表新增 `wechat_openid` 字段）。
    *   `UPDATE users SET wechat_openid = 'oCola7...' WHERE id = '当前登录用户ID';`

### 之后的扫码登录流程

1.  用户在登录页扫码。
2.  本平台告诉您：“来人是 OpenID 为 `oCola7...` 的人”。
3.  **查库**：您在自己的数据库查询：`SELECT * FROM users WHERE wechat_openid = 'oCola7...'`。
4.  **登入**：查到了关联的账号（比如 Admin），于是您让他在您的网站以 Admin 身份登录。

**总结**：
*   **本平台**提供“刷脸”（识别身份）。
*   **您的网站**负责“开门”（赋予权限/角色）。
